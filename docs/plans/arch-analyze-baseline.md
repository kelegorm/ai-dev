# arch-analyze — baseline (RED)

Что делает агент **без** скилла `arch-analyze`, получив flat-проект и
задачу «проверь архитектуру, всё ли хорошо».
Дата: 2026-05-22. Модель: sonnet. Проект: flat-фикстур `arch-migrate-fixture`.

## Вывод одной строкой

Агент делает **компетентный, но вольный** разбор: свой вердикт своими
словами, генерические советы, и **ноль знания о семействе `ai-dev`** —
о маркере, состояниях, эталоне, о том, какой скилл звать дальше. Скилл
должен не «научить смотреть на архитектуру» (агент это умеет), а
**структурировать** разбор и **встроить знание семейства**.

## Что агент сделал (сам, без скилла)

- Дал характеристику-вердикт («стартовый монолит в одной папке»).
- Разобрал: слои, доменную модель, слой данных, state management,
  навигацию, смешение data/domain, тесты, линт — широко и подробно.
- Выдал список рекомендаций по приоритетам.

## Расхождения с задуманным поведением (это и чинит скилл)

| Аспект | Сделал сам | `arch-analyze` требует |
|---|---|---|
| Состояние проекта | вердикт своими словами | классификация в одно из 5 именованных состояний (greenfield / flat / layered-no-marker / ai-dev-in-progress / ai-dev-settled) |
| Знание семейства | нет — не знает про `ai-dev` | назвать конкретный следующий скилл (`arch-bootstrap` / `arch-migrate` / `arch-audit`) |
| Маркер / ledger | не проверял | проверка маркера `<!-- ai-dev:arch-contract v1 -->` и ledger'а |
| Эталон | советует Clean-arch (`domain/data/presentation`) | сравнивать с эталоном (`app/ app_ports/ domain/ ex_systems/ ui/`) |
| Формат разбора | вольные 8 разделов + приоритеты 1-6 | структурированные пробы по сигналам, по одной строке-вердикту |
| Глубина | глубокий разбор + план улучшений | лёгкие пробы; глубокий разбор дрейфа — это `arch-audit` |

## Следствия для скилла (вход в Task 3, шаг 4)

`arch-analyze` обязан явно:

1. **Классифицировать в одно из 5 состояний** — не вольный вердикт.
2. **Знать семейство `ai-dev`**: проверять маркер/ledger, и в
   рекомендации называть конкретный следующий скилл.
3. **Сравнивать с эталоном** (`docs/reference-architecture/`), а не
   советовать произвольную хорошую архитектуру.
4. **Структурировать разбор** в фиксированный набор проб по сигналам с
   короткими вердиктами — а не свободные разделы.
5. **Не уходить в глубину** — лёгкие пробы, глубокий дрейф-разбор отдать
   `arch-audit`.

Базовую способность «оценить архитектуру» агент имеет — скилл фиксирует
**формат, состояния и встроенное знание семейства**.
